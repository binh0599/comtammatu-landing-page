## Critical Issues (Must Fix)

### Issue 1: Booking Modal — Missing Form Labels

**WCAG Criterion:** 1.3.1 — Info and Relationships (Level A)
**Severity:** Critical
**Location:** `src/components/BookingModal.tsx:55-124`

**Problem:**
All five form inputs (name, phone, date, time, guests) use `placeholder` text instead of proper `<label>` elements. Placeholders disappear on focus, leaving users with no persistent indication of what each field expects. Screen readers may not announce placeholders as labels.

**Impact:**
Screen reader users cannot identify form fields. Users with cognitive disabilities lose context when placeholders vanish. This fails Level A — the most basic compliance level.

**Remediation:**
Add visually hidden `<label>` elements for each input, or use `aria-label` attributes.

```tsx
// Before
<input type="text" placeholder="Họ và Tên" required ... />

// After
<label htmlFor="booking-name" className="sr-only">Họ và Tên</label>
<input id="booking-name" type="text" placeholder="Họ và Tên" required aria-label="Họ và Tên" ... />
```

Apply the same pattern to all five inputs (phone, date, time, select).

**Testing:** Tab through the form with VoiceOver/NVDA and verify each field is announced correctly.

### Issue 2: Booking Modal Backdrop — Click-to-Close Without Keyboard Equivalent

**WCAG Criterion:** 2.1.1 — Keyboard (Level A)
**Severity:** Critical
**Location:** `src/components/BookingModal.tsx:17-22`

**Problem:**
The modal backdrop closes the modal on click, but there is no keyboard handler (Escape key) to dismiss the modal. The close button exists but keyboard users who open the modal have no way to press Escape to close it. Additionally, focus is not trapped inside the modal — keyboard users can tab behind the modal overlay.

**Impact:**
Keyboard-only users may become stuck or unable to dismiss the modal. This is a keyboard trap risk.

**Remediation:**

```tsx
// Add Escape key handler and focus trap
useEffect(() => {
  if (!isOpen) return;
  const handleKeyDown = (e: KeyboardEvent) => {
    if (e.key === "Escape") setIsOpen(false);
  };
  document.addEventListener("keydown", handleKeyDown);
  return () => document.removeEventListener("keydown", handleKeyDown);
}, [isOpen]);
```

Additionally:
- Add `role="dialog"` and `aria-modal="true"` to the modal container.
- Add `aria-labelledby` pointing to the modal heading.
- Trap focus within the modal while open.
- Return focus to the trigger element on close.

### Issue 4: Custom Cursor Hides System Cursor

**WCAG Criterion:** 2.5.1 — Pointer Gestures (Level A) / 1.4.13 — Content on Hover or Focus (Level AA)
**Severity:** Critical
**Location:** `src/app/globals.css:44-56`, `src/components/ui/CustomCursor.tsx`

**Problem:**
The CSS sets `cursor: none !important` on all interactive elements for desktop devices. While the custom cursor is visually attractive, it hides the system cursor that users may depend on. Users with motor impairments using specialized cursor software, screen magnifiers, or high-contrast modes will lose their cursor entirely.

**Impact:**
Users relying on specialized cursor configurations or screen magnifiers cannot see the cursor. The custom cursor uses `mix-blend-exclusion` which can make it invisible against certain backgrounds.

**Remediation:**
Add a `prefers-reduced-motion` / user preference check, and provide a way to disable the custom cursor:

```css
/* Only hide cursor if user hasn't indicated accessibility needs */
@media (hover: hover) and (pointer: fine) and (prefers-reduced-motion: no-preference) {
  body:not(.a11y-cursor) { cursor: none !important; }
}
```

Better yet, make this opt-in rather than opt-out, or add a visible toggle in the UI.

### Issue 5: Mobile Menu — Not Accessible as Navigation Landmark

**WCAG Criterion:** 2.4.1 — Bypass Blocks (Level A) / 4.1.2 — Name, Role, Value (Level A)
**Severity:** Critical
**Location:** `src/components/Navbar.tsx:103-130`

**Problem:**
The mobile menu dropdown is a plain `<div>` with no `role`, `aria-expanded` state on the hamburger button, or `aria-controls` association. The hamburger button only has `aria-label="Menu"` but doesn't communicate whether the menu is open or closed.

**Impact:**
Screen reader users cannot determine if the menu is expanded or collapsed. The menu panel is not announced as navigation.

**Remediation:**

```tsx
// Hamburger button
<button
  onClick={() => setMobileMenuOpen(!mobileMenuOpen)}
  aria-label="Menu điều hướng"
  aria-expanded={mobileMenuOpen}
  aria-controls="mobile-nav"
  className="..."
>

// Mobile menu
<nav
  id="mobile-nav"
  role="navigation"
  aria-label="Menu chính (di động)"
  className={`... ${mobileMenuOpen ? "..." : "... hidden"}`}
>
```

### Issue 11: Scroll-Snap May Trap Keyboard Users

**WCAG Criterion:** 2.1.2 — No Keyboard Trap (Level A)
**Severity:** Serious
**Location:** `src/app/page.tsx:17`

**Problem:**
The `<main>` element uses `snap-y snap-mandatory`. With `snap-mandatory`, the scroll container forcibly snaps to sections, which can interfere with keyboard scrolling (arrow keys, Page Up/Down, Tab) and assistive technology scroll behavior. Users may become stuck on a section.

**Remediation:**
Consider using `snap-proximity` instead of `snap-mandatory`, or add `prefers-reduced-motion` media query to disable snap behavior:

```css
@media (prefers-reduced-motion: reduce) {
  main { scroll-snap-type: none !important; }
}
```

## Moderate Issues

### Issue 14: No `prefers-reduced-motion` Respect for Animations

**WCAG Criterion:** 2.3.3 — Animation from Interactions (Level AAA)
**Severity:** Moderate
**Location:** `src/app/globals.css` (all animation classes), `AnimationProvider.tsx`

**Problem:**
The site has extensive animations (fadeIn, slideIn, float, shimmer, pulseGlow, scrollDown, etc.) with no `prefers-reduced-motion` media query. Users who have indicated they prefer reduced motion will still see all animations.

**Remediation:**

```css
@media (prefers-reduced-motion: reduce) {
  .animate-fade-in, .animate-fade-in-up, .animate-fade-in-down,
  .animate-slide-left, .animate-slide-right, .animate-scale-in,
  .animate-float, .animate-shimmer, .animate-pulse-glow,
  .animate-scroll-down, .animate-border-glow, .animate-pop-drop {
    animation: none !important;
    transition: none !important;
    opacity: 1 !important;
    transform: none !important;
  }
  .reveal, .reveal-up, .reveal-left, .reveal-right,
  .reveal-scale, .reveal-fade, .section-reveal {
    opacity: 1 !important;
    transform: none !important;
    transition: none !important;
  }
}
```
